Fix SEC-01: Replace innerHTML-based label rendering with textContent to prevent DOM-based XSS.

Location: renderTransaction() function, specifically the tx-name div creation (line ~2950).

Current vulnerable code:
```javascript
<div class="tx-name">${tx.label_name||'Unlabeled'}</div>
```

Replace the entire row creation (lines ~2941-2953) with this pattern:
1. Keep the row creation as is: `const row = document.createElement('div');`
2. Build the inner HTML structure using createElement for each element instead of innerHTML template literal
3. For the tx-name div specifically:
   - Create: `const nameDiv = document.createElement('div');`
   - Set class: `nameDiv.className = 'tx-name';`
   - Set text safely: `nameDiv.textContent = tx.label_name || 'Unlabeled';`
   - Append to inner div

The inner.innerHTML should now contain ONLY static structural HTML with no user data interpolation. All user data (label_name, amount, time) set via textContent or as safe attributes.

Verify that ALL label_name references in renderTransaction() use textContent, not innerHTML. Search for other innerHTML instances in transaction rendering — ensure none interpolate tx.label_name or user-controlled data.

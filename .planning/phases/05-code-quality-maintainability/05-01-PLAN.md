<objective>
Extract the data layer (loadLabels, loadHistory, loadMonthTotals) into a DataModule factory function with encapsulated state. This establishes the modularization pattern and fixes global state fragility (QUAL-01).

Purpose: Remove direct access to global `allTxData` and `labels` variables. All reads/writes go through factory methods, enabling proper validation, caching, error handling, and event emission. This unblocks chart memoization (QUAL-04) which needs to listen to data-changed events.

Output: Single DataModule with public API (getLabels, getTransactions, loadLabels, loadHistory, addTransaction, updateTransaction, deleteTransaction, setLabels) and event emission (data-changed, labels-updated, cache-error). Global variables remain but only DataModule accesses them directly. Other code uses module methods.
</objective>
